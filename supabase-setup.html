1

Create a Supabase Project

Takes about 2 minutes. You need a free Supabase account.

Steps
1
Go to supabase.com → Start your project → sign in with GitHub
2
Click New project, name it stableos, pick a region close to your users (US East for Texas)
3
Set a strong database password and save it somewhere safe — you'll need it if you ever connect directly via psql
4
Wait ~2 min for provisioning, then go to Project Settings → API
5
Copy your Project URL and anon public key — you'll paste them into db.supabase.js in step 4
💡
The anon key is safe to expose in client-side code — it's locked down by Row Level Security. Never put the service_role key in your front-end HTML.

How to run
1
In your Supabase project, go to SQL Editor in the left sidebar
2
Click New query
3
Open supabase-schema.sql from your repo, paste its entire contents into the editor

Click Run (▶) — you should see "Success. No rows returned"

If you see an error like "relation already exists", the schema was partially run before. Click Table Editor to verify tables exist, then continue to step 3.
3

Configure Auth & Create Admin

Supabase Auth handles password hashing and sessions. You need to create your super admin account here.

Create your Super Admin account
1
In Supabase, go to Authentication → Users → click Add user
2
Enter stoneyw@gmail.com and a secure password, then click Create user
3
Copy the User UID shown in the users list (the long uuid like a1b2c3d4-...)
4
Go back to SQL Editor and run this, replacing the UUID with yours:
insert into platform_admins (auth_id, name, email)
values ('YOUR-AUTH-UUID-HERE', 'Stoney W.', 'stoneyw@gmail.com');
Auth settings to configure
1
Go to Authentication → Providers — make sure Email is enabled
2
Under Authentication → Email Templates, customize the confirmation email with your StableOS branding (optional)
3
Under Authentication → URL Configuration, add your GitHub Pages URL as a Redirect URL: https://stoneyw.github.io/StableOS/
4
Optionally, under Authentication → Settings, disable Confirm email during development so sign-ups work immediately
💡
When a new stable signs up via the onboarding page, db.supabase.js calls supabase.auth.signUp() which creates the Supabase Auth user automatically — you don't need to manually add each one.
4

Swap in the New Data Layer

Two edits to each HTML file — add the Supabase SDK and replace db.js with db.supabase.js.

1 — Add your credentials to db.supabase.js
1
Open js/db.supabase.js in your editor
2
Replace the two placeholder values at the top:
// js/db.supabase.js — lines 8–9
const SUPABASE_URL  = 'https://YOUR_PROJECT.supabase.co';
const SUPABASE_ANON = 'YOUR_ANON_KEY';

// Replace with values from:
// Supabase Dashboard → Project Settings → API
2 — Update your HTML files

In each HTML file, find the two script tags in the <head> and make these changes:

1
Add the Supabase SDK before your db script
2
Replace js/db.js with js/db.supabase.js
<!-- BEFORE -->
<script src="js/db.js"></script>

<!-- AFTER -->
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2/dist/umd/supabase.js"></script>
<script src="js/db.supabase.js"></script>
3 — Make your app async-aware

The new db.supabase.js returns Promises from every method. Update your React components to await them:

// OLD (synchronous — localStorage)
const horses = window.DB.getHorses();
setHorses(horses);

// NEW (async — Supabase)
useEffect(() => {
  window.DB.getHorses().then(h => setHorses(h));
}, []);

// OR with async/await in an effect:
useEffect(() => {
  (async () => {
    const horses = await window.DB.getHorses();
    setHorses(horses);
  })();
}, []);
💡
You can migrate one page at a time — keep db.js on pages you haven't migrated yet and use db.supabase.js on updated ones.
5

Push to GitHub & Go Live

Final checklist before your Supabase-backed StableOS goes public.

Pre-launch checklist
  • Supabase project created and schema SQL executed
  • Platform admin user created in Auth → Users and inserted into platform_admins
  • Project URL and anon key added to js/db.supabase.js
  • Supabase SDK script tag added to all HTML files
  • js/db.js replaced with js/db.supabase.js in all HTML files
  • React components updated to await all window.DB calls
  • GitHub Pages redirect URL added in Supabase Auth → URL Configuration
  • Tested sign-up flow end-to-end locally
  • Pushed all files to github.com/stoneyw/StableOS
Files to push to your repo
🗄️
supabase-schema.sql
Keep for reference / re-runs. Don't expose secrets here.
⚡
js/db.supabase.js
New async data layer. Contains your anon key (safe to expose).
📄
app.html → index.html
Rename so GitHub Pages serves it at the root URL.
🗑️
js/db.js
Can be removed once all pages are migrated.
